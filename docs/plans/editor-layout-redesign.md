# 編集画面レイアウトの見直し

## 目的・背景

現在の記事編集画面は「ルール」「英作文」「添削済み英文」「アドバイス」の 4 欄を 2×2 グリッドで均等に並べている。しかし実際の利用では:

- **ルールは編集時に見るものではない**: 記事作成の最初に設定したら、あとは添削時に AI へ渡されるだけ。常時一等地（左上 1/4）を占めるのは無駄
- 画面の意味構造は「**ユーザーの入力**」と「**AI のアウトプット**」の 2 グループなのに、現在のグリッドはそれが視覚的に読み取れない

## 対応方針

### 新レイアウト

```
+--------------------------------------------------------------+
| ← 一覧へ                                            ステータス |
+--------------------------------------------------------------+
| ▸ 文章生成のルール（折りたたみ・デフォルト閉）                   |
+--------------------------------------------------------------+
|  入力                        |  AI のアウトプット              |
| +--------------------------+ | +----------------------------+ |
| |                          | | | AI の添削済み英文           | |
| | ユーザーの英作文          | | +----------------------------+ |
| | （左カラム全高）          | | +----------------------------+ |
| |                          | | | AI からのアドバイス         | |
| +--------------------------+ | +----------------------------+ |
+--------------------------------------------------------------+
```

1. **ルールは折りたたみ化**: `<details>`（またはトグルボタン + パネル）でヘッダー直下に全幅配置し、**デフォルトで閉じる**。開いたときだけ textarea が現れる。ID・自動保存の配線（`input` イベント）は現状のまま
2. **本文エリアは左右 2 カラムのグループ構成**
   - 左カラム =「入力」グループ: ユーザーの英作文（カラム全高を使い、書くことに集中できる広さにする）
   - 右カラム =「AI のアウトプット」グループ: 添削済み英文（上）+ アドバイス（下）を縦に積む
   - 各カラムにグループ見出し（例: 「入力」「AI のアウトプット」）を付け、2 グループ構造を視覚化する。出力側は背景色を淡く変える等で読み取り専用であることも示す
3. **レスポンシブ**: 幅 900px 以下では現行同様に縦 1 カラム（ルール → 英作文 → 添削 → アドバイス）に落とす

### 実装ステップ

1. `article.html`: ルールを `<details class="rules">` に移し、`editor-grid` を「入力カラム」「出力カラム」の 2 カラム構造に組み替える
2. `style.css`: 折りたたみ・2 カラムグリッド・グループ見出し・出力側の配色を実装。既存の `.pane` スタイルは可能な範囲で流用
3. 動作確認

## 影響範囲

- `backend/public/article.html`: 構造変更（要素の入れ子・グループ化、`<details>` 追加）
- `backend/public/css/style.css`: `.editor-grid` まわりの書き換え + 追加スタイル
- `backend/public/js/article.js`: **変更なしの想定**。`rules` / `draft` / `corrected` / `advice` / `status` の ID を維持するため、自動保存・添削のロジックはそのまま動く
- バックエンド・API: 変更なし

## テスト方針

- `./run-server.sh` で起動し、ブラウザで編集画面を確認
  - ルールが折りたたまれた状態で開き、開閉できること。開いて編集すると自動保存 → 添削が走ること
  - 英作文の入力で従来どおり自動保存 → 添削が実行され、右カラムに結果が表示されること
  - ウィンドウ幅を狭めて 1 カラムに崩れること
- `npm run typecheck`（フロントは素の JS だが、念のためバックエンドに影響がないことの確認として）
