# 文章生成ルールを「AIルール」としてダイアログ形式にする

## 目的・背景

現在「文章生成のルール」はエディタ上部の `<details>` 折りたたみパネル。以下へ変更する。

- 名称を **「AIルール」** とし、ボタンを **AI のアウトプット側のグループ**（バイオレット系）に置く
- ボタンを押すとダイアログ（モーダル）が開き、そこでルールを編集する
- 記事の新規作成直後はダイアログを自動で開いておく
- 「ルールを入れると AI がそれに沿った添削・アドバイスをする」ことをダイアログ内に明記し、何のための入力欄なのか分かるようにする

インラインの折りたたみより画面がすっきりし、開いたときはモーダルなのでルール入力に集中できる。

## 対応方針

### UI（index.html / style.css）

- `<details class="rules-panel">` を廃止し、AI アウトプット列（`.column-output`）のグループ見出し直下に **「✎ AIルール」バーボタン** を置く
  - ルールが設定済みかどうかをボタンで可視化する（未設定なら「未設定」、設定済みならルールの先頭 1 行を薄字でプレビュー表示）。ダイアログ化で中身が見えなくなるデメリットをここで補う
  - スタイルは AI 出力ペインのヘッダーと同系（`--ai-head` 背景 + `--ai-border`）で、AI グループの一員であることを示す
- ネイティブ `<dialog>` 要素でモーダルを実装する（ビルドなし方針に合致。`showModal()` で Esc 閉じ・backdrop・フォーカストラップが標準で付く）
- ダイアログの構成
  - 見出し: 「AIルール」
  - 説明文: 「ここに書いたルールに沿って、AI が添削とアドバイスを行います。文体・レベル・語数などを自由に指定できます」
  - ルール入力の textarea（既存の `#rules` を移設。input → scheduleSave の既存 autosave はそのまま効く。placeholder に具体例を入れる）
  - 閉じるボタン（「閉じる」1 つ。保存は autosave 任せなので OK/キャンセルの 2 択にはしない）
- スタイル: `::backdrop` を半透明 slate、ダイアログはカードと同じ角丸 + シャドウ。見出しはバイオレット
- 既存の `.rules-panel.highlight` パルスアニメーションは削除（モーダル自体が十分目立つため不要）

### 挙動（app.js）

- ボタン click → `dialog.showModal()` + ルール textarea へフォーカス
- 新規作成ハンドラ: `selectArticle()` 後に同じ処理でダイアログを開く（前タスクの「開いておく」をダイアログ版に置き換え）
- ダイアログを閉じたとき（`close` イベント。Esc・閉じるボタン共通）:
  - 保留中の autosave があれば即時 flush → 添削実行。ルールを書き終えて閉じた瞬間に添削が走る体験にする
  - ボタンのプレビュー表示を更新する
- backdrop クリックでも閉じられるようにする（dialog 自身への click で `event.target === dialog` なら `close()`）
- モーダル表示中は背面操作がブロックされるため、「ダイアログを開いたまま別記事へ切り替え」の考慮は不要（前タスクで入れた selectArticle でのリセットは、ボタンプレビュー更新に置き換え）

## 影響範囲

- フロントエンドのみ（index.html / app.js / style.css）
- バックエンド・DB・API 変更なし
- 前タスク（rules-panel を開く + highlight）の実装は本タスクで置き換えられる

## テスト方針

- `npm run typecheck`
- ヘッドレス Chromium（playwright-core）
  - ボタン押下でダイアログが開き、textarea にフォーカス
  - 新規作成直後にダイアログが自動で開く
  - 説明文が表示されている
  - ルール入力 → 閉じる → 保存・添削が走る（API モックなしなら保存まで確認）
  - Esc / backdrop クリックで閉じる
  - 閉じた後、ボタンにルールのプレビューが反映される
  - スクリーンショットで見た目確認
