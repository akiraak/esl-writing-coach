# 文章生成ルールは新規作成直後は目立つように開いておく

## 目的・背景

「文章生成のルール」は `<details class="rules-panel">` で折りたたまれており、デフォルトは閉じている。
新規記事の作成直後はまずルールを設定してほしいのに、閉じたままだと存在に気づきにくい。
新規作成直後だけはパネルを開いた状態にし、目立つように軽くハイライトする。

## 対応方針

- `backend/public/js/app.js`
  - `rules-panel` の `<details>` 要素への参照を追加する
  - `selectArticle()` でルールパネルを閉じる（既存記事の選択時は従来どおり閉じた状態。新規作成時の開き状態が別記事へ持ち越されないようにする）
  - 新規作成ハンドラで `selectArticle()` 後にパネルを `open` にし、ルール textarea へフォーカスする（従来は本文へフォーカスしていたが、新規作成の最初の入力はルールなので変更）
  - パネルに一時クラス `highlight` を付与し、`animationend` で外す
- `backend/public/css/style.css`
  - `.rules-panel.highlight` にインディゴ系（入力色）の box-shadow パルスアニメーションを追加し、開いたことがひと目で分かるようにする

## 影響範囲

- フロントエンドのみ（`app.js` / `style.css` / 必要なら `index.html` に id 追加）
- バックエンド・DB 変更なし

## テスト方針

- `npm run typecheck`（backend、既存確認）
- ヘッドレス Chromium（playwright-core）で以下を確認
  - 新規作成 → ルールパネルが開いており、ルール欄にフォーカスがある
  - 既存記事を選択 → パネルは閉じている
  - 新規作成後に別記事へ切り替え → パネルが閉じる
