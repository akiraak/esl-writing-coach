# 新規作成ボタンの失敗を無言にしない

## 目的・背景

「新規作成ボタンが反応しない」という報告があった。調査の結果、ボタン・ハンドラ自体は
デスクトップ幅・モバイル幅とも正常動作（ヘッドレス Chromium で記事作成 → ルールダイアログ表示を確認）。
無反応に見える再現条件は **API サーバが落ちている / 到達できない** 場合で、現在の実装は
新規作成の fetch 失敗時に何も表示しないため、ユーザーからは「押しても何も起きない」ように見える。

## 対応方針

`backend/public/js/app.js` の新規作成ハンドラに try/catch と res.ok チェックを追加し、
失敗時は `setStatus('記事の作成に失敗しました', 'error')` で topbar のステータス欄に表示する。

## 影響範囲

フロントの新規作成ハンドラのみ。成功時の挙動（一覧更新 → 選択 → ルールダイアログ）は不変。

## テスト方針

ヘッドレス Chromium で POST /api/articles をブロックした状態でクリックし、エラーメッセージが
表示されることを確認。通常時の新規作成が従来どおり動くことも確認。
