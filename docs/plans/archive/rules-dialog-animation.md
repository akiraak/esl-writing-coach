# AIルールダイアログのサイズ調整と開くアニメーション

## 目的・背景

前回ほぼ全画面（ビューポート − 48px）にしたが大きすぎた。少し小さくする。
あわせて、開くときに「押したボタンから広がる」アニメーションを付け、どこから開いたのか分かるようにする。
新規作成時（自動オープン）も、AIルールボタンの場所が分かるようにボタン自体へアニメーションを入れる。

## 対応方針

### サイズ（style.css）

- `.rules-dialog` を `width: min(880px, calc(100vw - 80px))`・`height: min(660px, calc(100vh - 80px))` に縮小
  - 大画面では上限 880x660、小画面ではビューポート − 80px。textarea が残り全高を使う flex 構成は維持

### 開くアニメーション（app.js + style.css）

- `openRulesDialog()` で `showModal()` 直後に Web Animations API で再生:
  ボタン中心 → ダイアログ中心への translate + ボタン幅相当の scale から `transform: none` へ 240ms で展開
  （transform は開始位置の計算に実行時のボタン座標が要るため CSS でなく JS で行う）
- `::backdrop` は CSS の `[open]` アニメーションでフェードイン

### 新規作成時のボタン強調（app.js + style.css）

- 新規作成 → ダイアログ自動オープン（従来どおり。ボタンから広がるアニメーションで出どころを示す）
- ダイアログを閉じたタイミングで、AIルールボタンにバイオレットのリングを 2 回パルスさせる
  `.rules-button.pulse` アニメーションを再生（「ルールはここにある」と場所を教える）。`animationend` でクラス除去
- パルスは新規作成後の初回クローズのみ（フラグ管理。記事切り替えでフラグ解除）

## 影響範囲

- フロントエンドのみ（app.js / style.css）。HTML・バックエンド変更なし

## テスト方針

- ヘッドレス Chromium
  - ダイアログサイズが上限 880x660 に収まる
  - 開いた直後に transform アニメーションが走る（`getAnimations()` で確認）
  - 新規作成 → 閉じるとボタンに pulse クラスが付き、animationend で外れる
  - 既存記事でボタンから開閉した場合は pulse しない
  - スクリーンショットで見た目確認
