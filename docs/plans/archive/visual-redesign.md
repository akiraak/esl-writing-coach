# デザイン刷新（配色・質感・サイドバー開閉アニメーション）

## 目的・背景

現状の見た目が野暮ったい（灰色ベタ・境界が曖昧・アクセント色が弱い）。
また入力欄と AI 出力欄の区別が色で直感的に分からない。サイドバーの開閉も
display:none の切り替えで瞬間的に消えるため、アニメーションさせたい。

## 対応方針

### 配色・質感（style.css 全面刷新）

- ニュートラルは slate 系（#f1f4f9 / #e2e8f0 / #64748b / #1e293b）で統一し、CSS 変数で管理
- **入力系はインディゴ**（#4f46e5）: 新規作成ボタン・フォーカスリング・入力グループの見出し
- **AI 出力系はバイオレット**（#7c3aed）: 出力ペインの背景 tint・ヘッダー・グループ見出し
  → 入力と出力がひと目で区別できる
- ヘッダーにグラデーションのロゴマーク（✎）を追加、カード類は角丸 10px + 弱いシャドウ
- 記事一覧: ホバー・選択中（インディゴ tint + ボーダー）を明確化
- ペインは focus-within でインディゴのリングを表示

### サイドバー開閉アニメーション

- `.sidebar.collapsed { display: none }` をやめ、**width 300px → 0 のトランジション**（0.28s ease）に変更
- 中身が潰れないよう、サイドバーの子要素は幅を `--sidebar-w` に固定（overflow: hidden で切り取る）
- ディバイダーの矢印もホバー色をトランジション

## 影響範囲

- `backend/public/css/style.css` — 全面書き換え
- `backend/public/index.html` — ロゴマーク用 span 追加のみ
- JS・バックエンドは変更なし

## テスト方針

- ヘッドレス Chromium で開閉トグル（アニメーション後の最終状態）・リロード復元を確認
- 記事表示・未選択・折りたたみの各状態のスクリーンショットで配色とレイアウトを確認
