# サイドバー開閉ボタンを 2 ペインの境界に配置

## 目的・背景

現在、左ペイン（記事一覧）の折りたたみボタンはヘッダー左端の「☰」で、
どのペインを開閉するのか分かりにくい。開閉ボタンを左ペインと右ペインの
**境界（ディバイダー）** に置き、機能が一目で分かるようにする。

## 対応方針

- ヘッダーの「☰」ボタンを削除する
- `.layout` 内のサイドバーとエディタの間に、縦いっぱいの細いディバイダーボタン
  （`#sidebar-toggle`）を追加する
  - 開いているときは「◀」、折りたたみ時は「▶」を中央に表示
  - ホバーで背景・矢印色を強調してクリックできることを示す
- JS は `applySidebarCollapsed()` で矢印の向きを切り替える（ID は維持するため他は無変更）
- 折りたたみ状態の localStorage 保存は従来どおり

## 影響範囲

- `backend/public/index.html` — ☰ 削除、ディバイダーボタン追加
- `backend/public/js/app.js` — 矢印アイコンの切り替えを追加
- `backend/public/css/style.css` — `.sidebar-divider` スタイル追加、`.btn-icon` 削除
- バックエンドは変更なし

## テスト方針

- ヘッドレス Chromium で、ディバイダークリックによる開閉・リロード後の状態復元・
  矢印の向きをスクリーンショット付きで確認する
