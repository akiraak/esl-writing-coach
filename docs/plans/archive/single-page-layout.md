# 画面を 1 つにする（左ペイン記事一覧 + 右ペイン記事編集）

## 目的・背景

現在は「記事一覧（index.html）」と「記事編集（article.html）」の 2 画面をページ遷移で行き来している。
これを 1 画面に統合し、左ペインに記事一覧・右ペインに記事編集を常時表示することで、
記事の切り替えを高速にする。左ペインは折りたたんで執筆に集中できるようにする。

## 対応方針

### レイアウト

- `index.html` を単一画面に書き換える
  - ヘッダー: サイドバー折りたたみトグル / タイトル / 保存・添削ステータス
  - 本文: `display: flex` の 2 ペイン
    - **左サイドバー**（固定幅 ~300px）: 「新規作成」ボタン + 記事一覧（excerpt・更新日時・削除）
    - **右エディタ**（flex: 1）: 既存の編集画面（ルール折りたたみパネル + 入力/AI アウトプットの 2 カラム）をそのまま移植
- サイドバーはトグルボタンで折りたたみ（幅 0）。状態は `localStorage` に保存し再訪時に復元
- 記事未選択時は右ペインにプレースホルダ（「記事を選択するか新規作成してください」）を表示
- 900px 以下のレスポンシブは従来同様、エディタ内 2 カラムを縦積みにする

### JS

- `js/index.js` + `js/article.js` を統合した `js/app.js` を新規作成（旧 2 ファイルは削除）
- 選択中記事 ID は URL クエリ `?id=N` で保持（`history.replaceState`）。リロードで同じ記事を復元
- 記事切り替え時の整合性:
  - 保留中の autosave タイマーをクリアし、未保存変更があれば**切り替え前に旧記事を保存**してからロード
  - save / correction は実行時の記事 ID をキャプチャし、応答が返った時点で別記事に切り替わっていたら画面へ反映しない
- 保存完了時にサイドバーの一覧（excerpt・更新日時）を更新する
- 記事削除時: 選択中の記事を削除したら右ペインをプレースホルダに戻す

### 旧画面

- `article.html` は削除する（自分専用ローカルツールのため旧 URL の互換は持たない）

## 影響範囲

- `backend/public/index.html` — 全面書き換え
- `backend/public/article.html` — 削除
- `backend/public/js/app.js` — 新規（index.js + article.js を統合）
- `backend/public/js/index.js` / `js/article.js` — 削除
- `backend/public/css/style.css` — サイドバーレイアウト追加、不要スタイル（back-link 等）整理
- バックエンド（API・DB）は変更なし

## テスト方針

- `npm run typecheck`（バックエンド無変更の確認）
- サーバーを起動し、playwright-core のヘッドレス Chromium で以下を確認
  - 一覧が左ペインに表示され、クリックで右ペインに記事がロードされる
  - 新規作成 → 入力 → 自動保存 → 一覧の excerpt が更新される
  - サイドバーの折りたたみ / 展開、リロード後の状態復元
  - 記事削除（選択中記事の削除でプレースホルダ表示）
  - スクリーンショットでレイアウト崩れがないこと
